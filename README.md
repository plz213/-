# 今天吃什么 🍱

一款帮助你快速解决"今天吃什么"难题的微信小程序。随机推荐、条件筛选、历史记录、菜品收藏，让选择不再纠结。

---

## ✨ 功能特性

### 🏠 首页
- **每日推荐**：随机展示 3 道精选菜品，附带菜品图片
- **转盘抽选**：一键随机抽取今日菜品，告别选择困难症
- 确认后自动跳转至详情页查看菜品详细信息

### 🍳 尝尝鲜
- **条件筛选**：可按主食类型、食材、做法三类条件筛选
- **智能匹配**：根据筛选条件计算匹配度，优先推荐高匹配菜品
- **匹配度反馈**：完美匹配 / 不错的匹配 / 随机推荐 三档提示
- **历史记录**：每次抽取结果自动加入"最近吃过"列表

### 📋 回头客
- **最近吃过**：查看历史抽取记录，最多保存 50 条
- **我的收藏**：自定义添加喜爱的菜品，可上传菜品图片
- **标签切换**：两个标签页自由切换
- 登录后才能使用个性化数据功能

### 👤 个人中心
- **登录/退出**：通过微信授权登录，本地生成用户 ID
- **个人信息**：展示用户头像和昵称
- **清除缓存**：一键清除所有本地缓存数据
- **意见反馈**：反馈建议入口

### 📝 意见反馈
- **标题 + 详细描述**双字段输入反馈内容
- 提交后本地保存，显示成功提示

## 🛠 技术栈

| 技术 | 说明 |
|------|------|
| **微信小程序原生框架** | `.wxml` + `.wxss` + `.js` + `.json` |
| **本地存储 `wx.storage`** | 存储用户信息、历史记录、收藏菜品 |
| **ES6** | 箭头函数、解构赋值、模板字符串等 |
| **随机算法** | 菜品推荐加权随机抽取 |

### 关键实现
- **用户隔离**：每个用户使用 `user_${userId}_${dataType}` 作为存储 key，避免数据混淆
- **事件冒泡处理**：弹窗内部使用 `catchtap="stopPropagation"` 阻止点击外部关闭
- **筛选匹配度计算**：主食 +3 分，食材 +2 分，做法 +2 分，未选择项各 +1 分
- **加权随机**：按匹配度分数加权抽取菜品，分数越高被选中概率越大

## 🚀 快速开始

### 环境要求
- **微信开发者工具** v1.0 及以上
- **小程序基础库** v3.15.2 及以上
- AppID：`wx6e440c38bc5a277d`（可替换为你自己的 AppID）

### 运行步骤

1. **克隆仓库**
```bash
git clone <your-repo-url>
cd <project-name>
```

2. **导入项目**
   - 打开「微信开发者工具」
   - 点击「导入项目」
   - 选择项目目录：本项目根目录
   - 填入 AppID（或使用「测试号」）
   - 点击「导入」

3. **编译运行**
   - 开发者工具会自动编译，点击「编译」按钮
   - 模拟器中即可预览和测试

4. **真机调试**
   - 点击「真机调试」按钮
   - 使用微信扫码在手机上预览

## 🔧 核心数据结构

### 用户数据
```js
{
  isLogin: false,       // 登录状态
  userInfo: {           // 用户信息
    nickName: '微信用户',
    avatarUrl: '/images/头像.png'
  },
  userId: 'user_123...' // 本地生成的用户ID
}
```

### 菜品数据
```js
{
  name: '蛋炒饭',
  staple: '饭',
  ingredients: ['鸡蛋', '鸡肉'],
  methods: ['炒'],
  description: '经典蛋炒饭，配嫩滑鸡肉'
}
```

### 存储 Key
| Key | 说明 |
|-----|------|
| `token` | 模拟登录凭证 |
| `currentUserId` | 当前登录用户ID |
| `lastUserId` | 上次登录用户ID |
| `history_public` | 未登录用户历史记录 |
| `user_${id}_history` | 已登录用户历史记录 |
| `user_${id}_favorites` | 已登录用户收藏菜品 |

## 📝 使用说明

### 1. 首次使用
- 打开小程序后，点击「个人中心」→ 点击头像登录
- 授权微信获取用户信息（昵称、头像）
- 登录后即可使用收藏和历史记录功能

### 2. 快速选择
- **不确定吃什么？** → 首页点击「转盘抽选」按钮
- **有偏好？** → 进入「尝尝鲜」页面，选择主食/食材/做法后点击「开始抽取」
- **想自己点菜？** → 在「回头客」页面添加自己的收藏菜品

### 3. 管理收藏
- 在「回头客」→「我的收藏」标签页，点击「添加菜品」
- 填写菜品名称，上传菜品图片，选择相关分类
- 点击「确认添加」完成收藏
- 点击「取消收藏」可删除已收藏的菜品

## 🚫 注意事项

1. **本地存储限制**：微信小程序 `wx.setStorageSync` 单条数据最大 1MB，整体最大 10MB
2. **图片资源**：所有菜品图片为本地静态图片，不依赖网络请求
3. **模拟登录**：本项目使用模拟登录（`wx.getUserProfile` + 本地 ID 生成），**未接入真实后端**
4. **清除缓存**：在「个人中心」清除缓存后，所有历史记录和收藏数据会被清空，无法恢复
5. **打包发布**：`project.config.json` 已配置打包时忽略 `文档/` 目录和 `.docx` 文件

## 🛠 开发小贴士

### 新增菜品
在 `utils/foodLibrary.js` 中添加菜品对象，同时在 `images/food-image/` 放入对应图片，在 `utils/foodImages.js` 中注册图片路径映射即可。

### 修改主题色
在 `app.json` 中修改 `navigationBarBackgroundColor` 和 `tabBar.selectedColor`，在 `app.wxss` 中修改 CSS 变量即可全局替换主题。

### 更换 AppID
编辑 `project.config.json` 第 49 行的 `"appid"` 字段即可。
